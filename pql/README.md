# pql-test DAX Test Files

This directory holds per-report DAX test suites executed by `pql-test` via the XMLA endpoint.

## Structure

```
pql/
  <ReportName>.SemanticModel/
    DAXQueries/
      Schema.DEV.Tests.dax       ← auto-generated by npm run pql:generate — review and commit
      DataQuality.DEV.Tests.dax  ← auto-generated — commit as-is (key columns discovered from model)
```

## One-time setup (per machine)

```powershell
pip install --index-url https://test.pypi.org/simple/ --extra-index-url https://pypi.org/simple/ "pql-test==0.1.9"
pql-test auth login       # interactive browser — same account as your Power BI login
pql-test check-prereqs    # verifies ADOMD.NET is downloaded and ready
```

## Generating test files for a report

```powershell
npm run pql:generate
```

This reads `playwright/config/enterprise.generated.json`, calls the Power BI REST API
for each report's schema, and writes two fully auto-generated files:

- `Schema.DEV.Tests.dax` — existence assertions for every visible table and column
- `DataQuality.DEV.Tests.dax` — `ShouldBeDistinct` assertions for every column marked
  `IsKey=TRUE` in the model metadata.  No editing needed — key columns are discovered
  automatically from `INFO.COLUMNS()`.  If no key columns are found, the file is generated
  with a note explaining none were detected.

Review `Schema.DEV.Tests.dax` and trim any calculated/volatile tables not relevant for
drift detection.  Commit both files.

## Running checks

From `npm run setup` — select focus option [7] or [8] in the focus menu.
Results appear in the standard Playwright HTML report in `test-archive/<run-id>/`.
